ISSUE 1: Session Titles Not Auto-Generating
--------------------------------------------
Expected: First message becomes session title
Actual: Always shows "New Chat"

Root Cause: Two-part problem
  1. Edge Function title update is fire-and-forget (line 172-176)
     - Not awaited, may complete after stream closes
     - Title might not persist to database

  2. Chat.jsx never reloads sessions after message send
     - Sessions loaded once on mount (line 26-58)
     - useEffect dependency is [user], not [messages]
     - Even if title updates on server, client never sees it

Fix Strategy:
  - Edge Function: Add await to title update (move before streaming)
  - Chat.jsx: Reload sessions after message send completes
  - Or: Return updated session in response, update local state


ISSUE 2: Messages Disappearing on Session Switch
-------------------------------------------------
Expected: Send message in session A, switch to B, back to A, message is there
Actual: Message vanishes when switching away and back

Root Cause: Race condition + missing error handling
  1. Edge Function saves assistant message AFTER streaming completes (line 302-315)
     - Not awaited, message insert may still be pending
     - Stream closes before database write finishes

  2. Chat.jsx has no error handling for failed saves
     - Optimistic update shows message in UI immediately
     - If database save fails, message lost silently
     - No error displayed to user

Fix Strategy:
  - Edge Function: Move assistant message save BEFORE closing stream
  - Add await to message insert operation
  - Add error handling if insert fails
  - Chat.jsx: After streaming, reload messages to verify persistence
