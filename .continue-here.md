## Current Position

All three critical chat bugs have been identified, fixed, and tested locally via Playwright. Fixes are committed locally and ready for deployment.

### What Was Done

1. **Investigated three chat issues** reported by user (all supposedly still broken after prior fixes)
   - Issue 1: Streaming text arrived as huge blocks instead of flowing
   - Issue 2: Session titles never updated from "New Chat"
   - Issue 3: Messages disappeared when switching sessions

2. **Found actual root causes** (not what was previously diagnosed):
   - Issue 1: `setTimeout(0)` was too aggressive - React still batches events arriving within 10ms
   - Issue 2: `reloadSessions()` call in finally block was **overwriting** the optimistic title update with DB data (null titles due to replication lag)
   - Issue 3: Silent `.catch()` errors were hiding save failures

3. **Applied three targeted fixes:**
   - Issue 1: Increased delays (0ms → 5ms frontend, 5ms → 10ms backend)
   - Issue 2: Removed `reloadSessions()` call entirely, kept optimistic update
   - Issue 3: Changed `.catch()` to destructured error handling, ensured correct sessionId used in message reload

4. **Local testing via Playwright verified:**
   - ✓ Streaming: 3+ DOM updates during response (confirmed incremental rendering)
   - ✓ Title: Optimistic update works immediately (verified in browser logs)
   - ✓ Persistence: Messages persist across session switches
